# Migrating from OACP v0.1 to v1.0

**Audience:** integrators on `@oacp/sdk` / `oacp-sdk`, MCPLab Docker users, and anyone still calling legacy playground APIs.

OACP **v1.0** (Day 54 freeze) stabilizes the protocol message `version`, published SDK semver, and the `/v1/observability/*` Console API. The legacy inline playground is deprecated; the **OACP Console** (`/console`) is the product surface.

---

## Summary

| Area                       | v0.1 (alpha)               | v1.0 (frozen)                                                |
| -------------------------- | -------------------------- | ------------------------------------------------------------ |
| Protocol `version` field   | `"0.1"`                    | `"1.0"` (preferred); `"0.1"` still accepted during migration |
| `@oacp/core` / `@oacp/sdk` | `0.1.x`                    | `1.0.0`                                                      |
| `oacp-sdk` (Python)        | `0.1.x`                    | `1.0.0`                                                      |
| Console snapshot           | `GET /playground/snapshot` | **`GET /v1/observability/snapshot`**                         |
| OpenAPI                    | —                          | `GET /v1/openapi.json`                                       |
| Legacy snapshot            | Primary in playground era  | **Deprecated** — removed Day 60                              |

## Protocol messages

### Bump the `version` field

All outbound OACP messages should use `"version": "1.0"`:

```json
{
  "type": "task_request",
  "version": "1.0",
  "message_id": "…",
  "trace_id": "…",
  "from": "agent://coordinator",
  "timestamp": "2026-07-01T12:00:00.000Z"
}
```

**SDKs:** use `PROTOCOL_VERSION` from `@oacp/core` or `oacp_sdk.defaults` — defaults to `1.0` in v1.0 packages.

**Read compatibility:** the reference server validator still accepts `"0.1"` messages until you migrate senders. New code should emit `1.0` only.

### Schema `$id` URLs

Canonical JSON Schemas moved from `https://oacp.dev/schemas/v0.1/` to `https://oacp.dev/schemas/v1.0/`. Message shapes are unchanged for the four core types (`task_request`, `task_response`, `delegation`, `capability_query`).

## HTTP observability API

### Replace legacy snapshot polling

| Legacy (deprecated)                         | v1 replacement                                    |
| ------------------------------------------- | ------------------------------------------------- |
| `GET /playground/snapshot?trace_id=&limit=` | `GET /v1/observability/snapshot?trace_id=&limit=` |

The legacy route returns **identical JSON** but includes deprecation headers:

- `Deprecation: @1754006399`
- `Sunset: Wed, 31 Jul 2026 23:59:59 GMT`
- `Link: </v1/observability/snapshot>; rel="successor-version"`

**Removal:** Day 60 (v1.0.0 launch week). Update clients before then.

### `@oacp/observability-client`

Default path is already `/v1/observability/snapshot`. Legacy fallback to `/playground/snapshot` on `404` remains for older Docker images until you upgrade the OACP server image.

```typescript
import { createObservabilityClient } from '@oacp/observability-client';

const client = createObservabilityClient({
  baseUrl: 'http://127.0.0.1:3000',
  // legacyFallback: true  // default — disable once server is v1.0+
});
```

### Python observability helpers

`oacp_sdk.observability.fetch_snapshot()` prefers v1 and falls back to legacy on `404` when `require_v1=False` (default).

### API key auth (Day 52)

When `OACP_API_KEY` is set on the server, send:

- Header: `X-Api-Key: <secret>` or `Authorization: Bearer <secret>`
- SSE: `?api_key=<secret>` on `/v1/observability/events` (browser `EventSource`)

Public: `GET /v1/observability/runtime-config` (no key required).

## Package upgrades

### TypeScript monorepo

```bash
pnpm update @oacp/core@1.0.0 @oacp/sdk@1.0.0 @oacp/observability-client@1.0.0
```

Workspace consumers: bump `workspace:*` packages and run `pnpm build`.

### Python

```bash
pip install "oacp-sdk>=1.0.0,<2"
```

## MCPLab Docker

1. Pull / rebuild the OACP image that includes v1 routes and Day 53 persistence.
2. Point Console proxy at the OACP server (`:3847` unified stack or `:3001` MCPLab sidecar).
3. Verify: `curl -s http://127.0.0.1:3847/v1/openapi.json | jq .info.version` → `"1.0.0"`.
4. Old crew run links: startup sync (Day 53) backfills traces after container recreate.

---

## Playground redirect

`GET /playground` → `302` `/console` (unchanged). Do not embed new features on playground HTML — it is frozen.

---

## Breaking change policy after v1.0

- **`/v1/observability/*`:** breaking response schema changes require bumping `info.version` in `specs/openapi/v1.json` and updating `specs/openapi/v1.lock.json` (CI enforced).
- **Protocol:** breaking message schema changes require protocol version `1.1` or `2.0` per [protocol-spec.md](../protocol-spec.md).
