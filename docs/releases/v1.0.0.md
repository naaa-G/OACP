## Summary

OACP v1.0 freezes the multi-agent protocol, reference server `/v1/observability/*` API, TypeScript and Python SDKs at semver **1.0.0**, and ships the **OACP Console** as the primary observability product. MCPLab remains the flagship MCP Ă— OACP demo; legacy playground snapshot is deprecated.

## Upgrade from v0.1

See **[migration guide](../migration/v0.1-to-v1.0.md)** â€” key moves:

1. Use `GET /v1/observability/snapshot` instead of `/playground/snapshot`
2. Set message `version` to `"1.0"` (SDKs do this automatically)
3. Bump packages to `1.0.0`

## API freeze enforcement

- Spec: [`specs/openapi/v1.json`](../../specs/openapi/v1.json)
- Lock: [`specs/openapi/v1.lock.json`](../../specs/openapi/v1.lock.json)
- CI: `node scripts/verify-api-freeze.mjs`
- Live validation: `server/tests/openapi-freeze.test.ts` compiles OpenAPI response schemas with AJV and asserts snapshot, runtime-config, trace graph, import, and legacy snapshot bodies match the frozen contract.

Breaking `/v1/` response changes without bumping `info.version` fail CI.
